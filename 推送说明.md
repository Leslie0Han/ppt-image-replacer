# 推送到 GitHub 说明

## 方法一：使用 Personal Access Token（推荐）

### 步骤 1：创建 GitHub 仓库

1. 访问 https://github.com/new
2. 仓库名称：`ppt-image-replacer`
3. 描述：`A visual tool for batch replacing images in PowerPoint files`
4. 选择 Public
5. **不要**勾选 "Add a README file"、"Add .gitignore" 或 "Choose a license"
6. 点击 "Create repository"

### 步骤 2：生成 Personal Access Token

1. 访问 https://github.com/settings/tokens
2. 点击 "Generate new token (classic)"
3. Note：填写 "PPT Image Replacer"
4. Expiration：选择 "No expiration" 或根据需要选择
5. 勾选 "repo" 权限
6. 点击 "Generate token"
7. **复制生成的 token**（只显示一次）

### 步骤 3：运行推送脚本

双击 `push_with_token.bat`，按提示输入：
- GitHub 用户名
- Personal Access Token
- 仓库名称（默认 `ppt-image-replacer`）

## 方法二：手动推送

### 步骤 1：配置 Git

```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

### 步骤 2：添加远程仓库

```bash
git remote add origin https://github.com/lesliehanx/ppt-image-replacer.git
```

### 步骤 3：推送

```bash
git branch -M main
git push -u origin main
```

## 方法三：使用 GitHub CLI

### 步骤 1：安装 GitHub CLI

访问 https://cli.github.com/ 下载并安装

### 步骤 2：登录

```bash
gh auth login
```

### 步骤 3：创建仓库并推送

```bash
gh repo create ppt-image-replacer --public --source=. --remote=origin --push
```

## 推送后

### 创建 Release

1. 访问 https://github.com/lesliehanx/ppt-image-replacer/releases
2. 点击 "Create a new release"
3. 标签版本：`v1.0.0`
4. 发布标题：`PPT Image Replacer v1.0.0`
5. 描述：
   ```
   ## 功能特点
   - 可视化界面，按页分组显示图片
   - 支持单张替换、多选替换和批量替换
   - 支持扫描组合形状中的图片
   - 保持图片位置和大小不变
   ```
6. 点击 "Publish release"

### 发布到 PyPI（可选）

如果想让用户可以通过 `pip install` 安装：

1. 注册 PyPI 账号：https://pypi.org/account/register/
2. 安装打包工具：
   ```bash
   pip install build twine
   ```
3. 构建包：
   ```bash
   python -m build
   ```
4. 上传：
   ```bash
   twine upload dist/*
   ```

## 常见问题

### Q: 推送时提示 "Permission denied"
A: 检查 Personal Access Token 是否正确，是否有 repo 权限

### Q: 推送时提示 "Repository not found"
A: 确保仓库已创建，且名称正确

### Q: 如何更新代码？
A: 
```bash
git add .
git commit -m "Update description"
git push
```

### Q: 如何创建新版本？
A:
```bash
# 修改版本号后
git add .
git commit -m "Bump version to 1.1.0"
git tag v1.1.0
git push origin main --tags
```
